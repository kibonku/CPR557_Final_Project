OVERVIEW
--------
An interactive 3D plant modeling application built on Vulkan.
Demonstrates Bezier surfaces, surface-of-revolution generation,
hierarchical scene graph (with parent-child transform propagation),
corrected camera navigation, real-time wind/light response, and
interactive Bezier surface editing.

COMPUTER GRAPHICS TECHNOLOGY USED
----------------------------------
- Vulkan rendering pipeline (vertex/fragment shaders, descriptor sets,
  push constants, swap chain, depth buffering)
- Bezier curve evaluation using Bernstein basis (de Casteljau algorithm)
- Surface of revolution: 3D geometry generated by revolving a 2D Bezier
  profile around the X-axis
- Analytic surface normals (computed from 2D tangent and revolution angle)
- UV coordinates stored per vertex (ready for texture mapping)
- Hierarchical scene graph with parent-child transform composition:
    plant_root → pot, stem, leaves_group → leaf1, leaf2, leaf3
  Parent transforms propagate to all descendants via recursive traversal.
- DFS scene graph traversal (HW3 pattern) for rendering and selection
- Diffuse + ambient Phong lighting model (UBO-based, GPU-computed)
- Camera model with orbit, pan, zoom, twist, and fit-all (HW4 pattern)
- Mode switching between interactive edit mode and 3D navigation mode

BUILD INSTRUCTIONS
------------------
Mac/Linux:
  bash compile-unx.bat

Windows:
  compile-win.bat

Requires: Vulkan SDK, GLFW, GLM installed and on PATH.

KEY COMMANDS
------------
MODE SWITCHING
  SPACE           Toggle Edit Mode / Navigation Mode

NAVIGATION MODE
  R + mouse drag  Rotate camera around scene
  P + mouse drag  Pan camera (translate)
  Z + mouse drag  Zoom in / out
  T + mouse drag  Twist (roll) camera
  F               Fit-all (reset camera to show full scene)
  Arrow keys      Rotate camera (no mouse needed)
  Shift + arrows  Pan camera (no mouse needed)
  = / +           Zoom in
  - / _           Zoom out
  C               Toggle perspective / orthographic projection

WIND / LIGHT RESPONSE (Navigation Mode)
  W + mouse move  Point mouse to set wind direction.
                  Leaves bend toward the mouse position.
                  Move mouse left/right/up/down to change wind direction.
                  Release W to hold the current wind.

SCENE GRAPH
  N               Select next node (cycles through all nodes in DFS order)
                  Selected node spins visually as confirmation.
  Shift+N         Show / hide surface normals on the interactive surface
  Tab             Print current scene graph to terminal

EDIT MODE (Interactive Bezier Surface)
  Left click      Place a Bezier control point (upper half of screen only)
  B               Build revolution surface from current control points
  M               Reset / clear all control points and surface
  X               Toggle helical twist on the revolution surface
  G               Toggle rainbow color gradient

STRETCH GOAL — Texture Mapping
  Texture mapping is not included in this build. UV coordinates are
  stored per vertex and ready for a sampler descriptor extension.

BUGS / LIMITATIONS
------------------
- Bezier control points must be placed in the upper half of the screen
  (y < 0.5 in normalized coordinates) in edit mode; clicks in the
  lower half are rejected (the axis of revolution occupies that space).
- The wind direction is set continuously while W is held. There is no
  smooth interpolation between wind states.
- Texture mapping is listed as a stretch goal and is not implemented.

POTENTIAL FUTURE IMPROVEMENTS
------------------------------
- Texture mapping: a sampler descriptor and PNG loader would complete
  the stretch goal (UVs are already computed).
- Animated Bezier control points: moving control points over time would
  create organic leaf-growth animations.
- Multiple pre-defined plant profiles selectable at runtime.
- Export scene graph to JSON for save/load.

ACHIEVEMENT OF STRETCH GOALS
-----------------------------
- Texture mapping: NOT implemented (UV coordinates stored, ready to add).
