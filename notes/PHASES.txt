# Phase 1: Hybrid Scene Construction. (Completed)
Initialize a 3D environment containing a pre-constructed plant model (stem and leaves) built via Bezier logic. 
This phase establishes the base coordinate system and rendering pipeline for the "pre-built" components of the scene.

# Phase 2: Interactive Surface of Revolution (The Pot). (Completed)
Implement an Edit Mode (toggled by SPACE) allowing for real-time user input. 
The user defines 5–6 control points to create a Bezier profile. 
This profile is then used to generate a 3D pot via a surface-of-revolution algorithm, triggered by the B key.

# Phase 3: Hierarchical Scene Graph Organization. (Completed)
Organize all components—the pre-built plant and the live-generated pot—into a hierarchical scene graph. 
This ensures that the plant remains correctly seated within the pot regardless of global transformations.

# Phase 4: Advanced Camera Navigation. (Completed)
Integrate a robust camera system supporting rotation, panning, zooming, and twisting. 
Includes a "fit-all" view to automatically frame the plant and pot within the viewport.

# Phase 5: Environmental Response (Wind/Light). (Completed)
Implement real-time leaf response to environmental variables. 
Using the W key, the user simulates wind direction, which triggers procedural bending or rotation of the leaf nodes via the scene graph.

# Phase 6: Automated Bezier Control-Point Animation. (Completed)
Incorporate dynamic leaf movement through the _animateLeaves function. 
This moves specific Bezier control points over time, resulting in a more fluid, organic bending motion compared to simple rigid-body transformations.

# Phase 7: Stretch Goal – Texture Mapping. (Completed)
Apply textures to the live-built pot and pre-built leaves to enhance visual fidelity. 
This remains a secondary priority to ensure the stability of the live-generation logic.
